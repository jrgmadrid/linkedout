# linkedout

chrome extension (manifest v3) that hides promoted posts, social-graph noise, and reposts from the linkedin feed. anything filtered collapses into a native-looking "hidden post" card that says why, with a per-post **show** button to bring it back (the card just shrinks to a slim "post shown" bar with a **hide** button, so it's reversible either direction).

click the toolbar icon to pick what gets filtered:

- **promoted posts** (on by default)
- **reaction spillover** — "jane doe likes this"
- **comment spillover** — "jane doe commented on this"
- **reposts** — "jane doe reposted this"
- **fig-leaf reposts** (on by default) — reposts whose only commentary is "couldn't agree more"
- **follow spillover** — "jane doe follows acme corp"
- **suggested posts** — "recommended for you"
- **news & games** — the "linkedin news" / "today's puzzles" rail widgets
- **slop chips** (on by default) — engagement-bait posts stay visible but get per-family verdict chips; see below
- **chip hide filters** (off by default) — each chip (broetry, engagement bait, ad spam, undisclosed promo, certified slop) can also collapse its posts behind a placeholder; judge-minted chips collapse once the verdict lands

## how it works

detection is calibrated against linkedin's 2026 frontend (hashed css-module classes, `componentkey` attributes), harvested from a live feed — see `harvest.js`, a debug exporter kept around for re-tuning when linkedin inevitably moves things.

- post root: `div[role="listitem"][componentkey*="FeedType"]`, with the classic-dom selectors kept as fallback (linkedin a/b tests its rendering stacks).
- promoted posts: `[aria-label="View Sponsored Content"]` (an accessibility disclosure) or a standalone "promoted" label leaf, exact-matched against a handful of locale strings.
- socially-surfaced posts: the surfacing header ("jane doe commented") is always the first text in the post, before the author block — patterns only match against that head segment, so body text can't false-positive.

a content script watches the feed with a `MutationObserver` (linkedin's an spa, so posts stream in forever) and classifies each one exactly once into a `data-dp-reasons` attribute. the enabled filter set is written to a `data-dp-hide` attribute on `<html>`, and css rules generated by `content.js` from `DP_FILTERS` intersect the two — so toggling a filter in the popup applies instantly, both directions, with no re-scan. settings sync via `chrome.storage.sync`.

## slop chips

posts get scored for *slop* — formulaic engagement-farming prose. it's an aesthetic call about the text, deliberately **not** an "ai-generated" accusation: a human-written hustle post earns its chips just fine. signals sort into three families, each rendering its own chip when it fires: **broetry** (line-stacking, "it's not x, it's y", em-dashes, buzzwords), **engagement bait** ("agree?", repost ctas, hashtag walls, reach hacks like "link in the first comment", 𝗳𝗮𝗸𝗲-𝗯𝗼𝗹𝗱 unicode styling), and **ad spam** (all-caps shouting, link piles, contact blocks). pseudo-bold text gets folded back to ascii before scoring, so nobody smuggles buzzwords past the lexicons in fancy unicode. a judge-confirmed post gets a filled **certified slop** chip. hover any chip for its itemized offense receipt.

a fourth chip, **undisclosed promo**, is minted only by the judge: promotion dressed up as personal content or free value. regex never convicts on intent — cheap promo markers ("register now", discount codes, launch phrasing) just send a suspected post to the judge, who lets honest promotion walk. job posts, launch announcements, and event invites that say what they are keep their dignity. the verdict's independent of the slop score, so a well-written stealth ad wears the chip anyway.

two tiers, two brains:

1. **heuristic scorer** (`slop.js`) — free, local, synchronous; runs on every post's commentary. each family chips independently at its own points threshold; clearly-fine posts are done here.
2. **llm judge** (`worker.js`) — posts whose aggregate score lands in the ambiguous middle band (or that trip a promo marker below it) go to an openai-compatible endpoint (deepseek by default) from the background service worker. the judge can certify (filled chip) or clear the heuristic chips outright — the mercy path for sincere posts that just look like slop. verdicts cache per post, judge at most once, and every failure mode falls back to the heuristic chips. the feed never waits on the network.

the judge is optional and byok. hit **judge key** in the popup footer (or right-click the toolbar icon → options) and paste an api key — it lands in `chrome.storage.local`, machine-local, never synced, never readable by the page. the options page has a **test the judge** button that fires one real call, because fail-open means a broken key is otherwise silent. for development there's also `config.local.example.js` → `config.local.js` (gitignored); a ui-entered key wins over the file. without either, the heuristic tier runs alone and the worker answers every request with a shrug. any openai-compatible endpoint works via the base-url field, but a non-deepseek host also needs adding to `host_permissions` in `manifest.json` — chrome won't let the worker call origins it never declared. scoring weights are calibrated against a labeled post set — run `node test/slop.test.js`.

## install

1. open `chrome://extensions`
2. flip on **developer mode** (top right)
3. **load unpacked** → point it at this directory

after edits, hit **reload** in the popup footer (or the ↻ on the extension card) — either way the extension refreshes its own linkedin tabs, so the new code is live without touching the browser.

## caveats

- purely visual. linkedin still counts the impression server-side; you're just not subjected to it.
- header pattern-matching is english-only (promoted-label detection isn't). using linkedin in another language means the social filters won't match until you add their patterns to `DP_FILTERS` in `filters.js`. the slop scorer's signal lexicons are english-only too.
- "reposts" covers both header-style reposts ("… reposted this") and the bare ones the new ui renders as an embedded original with no header (caught by a second "Visibility:" disclosure plus back-to-back timestamps with no commentary between). a repost where someone actually wrote something gets left alone — unless the "something" is "couldn't agree more", which is what the fig-leaf filter is for.
- everything's additive, so if linkedin reshuffles the dom the worst case is a filter quietly stops matching and nothing breaks. re-run "export feed markers" (options page, under maintenance) to harvest fresh signals and re-tune.
- this readme uses em-dashes, buzzword-adjacent phrasing, and at least one "it's not x, it's y". run it through the scorer and it earns a chip. i'm aware. the scorer is right.
