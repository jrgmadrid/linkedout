# Better LinkedIn

> Working title. Formerly "LinkedIn De-Promoter" — renamed when the scope grew
> beyond ad-blocking (spillover filters, repost detection, more coming).

Chrome extension (Manifest V3) that hides promoted posts, social-graph noise,
and reposts from the LinkedIn feed. Filtered posts collapse into a native-
looking "Hidden post" card showing the reason, with a per-post **Show**
button to reveal (the card stays as a slim "Post shown" bar with a **Hide**
button, so it's reversible). Click the toolbar icon to pick filters:

- **Promoted posts** (on by default)
- **Reaction spillover** — "Jane Doe likes this"
- **Comment spillover** — "Jane Doe commented on this"
- **Reposts** — "Jane Doe reposted this"
- **Follow spillover** — "Jane Doe follows Acme Corp"
- **Suggested posts** — "Recommended for you"

## How it works

Detection is calibrated against LinkedIn's 2026 frontend (hashed CSS-module
classes, `componentkey` attributes), harvested from a live feed — see
`harvest.js`, a debug exporter kept around for re-tuning when LinkedIn shifts.

- Post root: `div[role="listitem"][componentkey*="FeedType"]`, with the
  classic-DOM selectors kept as fallback (LinkedIn A/B tests rendering stacks).
- Promoted posts: `[aria-label="View Sponsored Content"]` (an accessibility
  disclosure) or a standalone "Promoted" label leaf, exact-matched against
  several locale strings.
- Socially-surfaced posts: the surfacing header ("Jane Doe commented") is
  always the first text in the post, before the author block — patterns are
  matched only against that head segment, so body text can't false-positive.

A content script watches the feed with a `MutationObserver` (LinkedIn is an
SPA, so posts stream in continuously) and classifies each post exactly once
into a `data-dp-reasons` attribute. The set of enabled filters is written to a
`data-dp-hide` attribute on `<html>`, and CSS rules generated by `content.js` from `DP_FILTERS` intersect the two —
so toggling a filter in the popup applies instantly, in both directions,
without re-scanning anything. Settings sync via `chrome.storage.sync`.

## Install

1. Open `chrome://extensions`
2. Enable **Developer mode** (top right)
3. **Load unpacked** → select this directory

Reload the extension after any edits (the ↻ button on the extension card),
then refresh linkedin.com.

## Caveats

- Purely a visual filter — LinkedIn still counts impressions server-side.
- Header pattern-matching is English-only (promoted-label detection is not).
  Using LinkedIn in another language means the social filters won't match
  until their patterns are added to `DP_FILTERS` in `filters.js`.
- "Reposts" covers both header-style reposts ("… reposted this") and bare
  reposts the new UI renders as an embedded original with no header (detected
  by a second "Visibility:" disclosure plus back-to-back timestamps with no
  commentary between). A repost where someone added their own commentary is
  left alone — that's usually the desirable behavior.
- Everything is additive, so if LinkedIn shifts the DOM again the worst case
  is a filter silently stops matching — nothing breaks. Re-run the popup's
  "Export feed markers" to harvest fresh signals and re-tune.
